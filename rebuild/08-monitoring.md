# 08 — Monitoring (Beszel)

Beszel is a lightweight server monitoring dashboard. It uses a **hub + agent** architecture: the hub is the web UI that aggregates metrics; the agent runs on the host and streams data to it. Both are deployed in a single stack.

The hub and agent are configured with a shared key and token generated by the hub — so the setup is a two-pass deploy: start the hub first, get the credentials, then update the stack with those credentials so the agent can connect.

**Prerequisites:**

- Guide 06 complete — NPM running, wildcard cert imported
- Docker network `proxy` created

---

## Section 1: First Deploy (Hub Only)

Stack name: `beszel`

In Portainer, create a new stack named `beszel`. Paste the full compose content below, then add the environment variables with placeholder values for now — you'll update them after the hub generates the real credentials.

```yaml
services:
  beszel:
    image: henrygd/beszel
    container_name: beszel
    restart: unless-stopped
    environment:
      - APP_URL=http://monitor.in.alybadawy.com:8090
    networks:
      - proxy
    volumes:
      - beszel_data:/beszel_data

  beszel-agent:
    image: henrygd/beszel-agent
    container_name: beszel-agent
    restart: unless-stopped
    network_mode: host
    volumes:
      - /var/run/docker.sock:/var/run/docker.sock:ro
      - ./beszel_agent_data:/var/lib/beszel-agent
      # Monitor additional disks by mounting a path under /extra-filesystems:
      # - /mnt/disk/.beszel:/extra-filesystems/sda1:ro
    environment:
      LISTEN: 45876
      KEY: ${BESZEL_KEY}
      TOKEN: ${BESZEL_TOKEN}
      HUB_URL: http://monitor.in.alybadawy.com:8090

volumes:
  beszel_data:

networks:
  proxy:
    external: true
```

In the **Environment variables** section, add placeholder values:

| Variable       | Value       |
| -------------- | ----------- |
| `BESZEL_KEY`   | `placeholder` |
| `BESZEL_TOKEN` | `placeholder` |

Click **Deploy the stack**. The hub will start; the agent will fail to connect until the real credentials are set — that's expected at this stage.

## Section 2: NPM Proxy Host

Before logging into the hub, set up the NPM proxy so the `APP_URL` resolves correctly.

1. **Proxy Hosts → Add Proxy Host**
2. **Details** tab:
   - **Domain Names:** `monitor.in.alybadawy.com`
   - **Scheme:** `http`
   - **Forward Hostname/IP:** `beszel`
   - **Forward Port:** `8090`
   - **Block Common Exploits:** On
3. **SSL** tab:
   - **SSL Certificate:** `wildcard-in-alybadawy-com`
   - **Force SSL:** On
   - **HTTP/2 Support:** On
4. Click **Save**

Test: open `https://monitor.in.alybadawy.com` in your browser.

---

## Section 3: Initial Setup and Credential Generation

1. Open `https://monitor.in.alybadawy.com`
2. Create an admin account and log in
3. Go to **Systems → Add system**
4. Fill in:
   - **Name:** `lab`
   - **Host / IP:** `172.20.20.5`
   - **Port:** `45876`
5. Beszel displays a **public key** and a **token** — copy both

---

## Section 4: Update the Stack with Real Credentials

In Portainer, go to **Stacks → beszel → Environment variables**. Update both values with the credentials from Section 3:

| Variable       | Value                                   |
| -------------- | --------------------------------------- |
| `BESZEL_KEY`   | *(public key from Section 3, step 5)*   |
| `BESZEL_TOKEN` | *(token from Section 3, step 5)*        |

Click **Update the stack**. Portainer will restart the agent with the correct credentials.

---

## Section 5: Verify the Connection

In the Beszel hub UI, the `lab` system should flip to **Connected** within a few seconds and begin showing live metrics.

If it stays disconnected:

```bash
# Check agent logs
docker logs beszel-agent

# Confirm port 45876 is bound on the host
ss -tlnp | grep 45876
```

## Troubleshooting

### Hub shows system as disconnected after credentials update

```bash
docker logs beszel-agent
ss -tlnp | grep 45876
```

- If port 45876 is not listed: agent failed to start — verify `BESZEL_KEY` and `BESZEL_TOKEN` are set correctly in the stack env vars
- The key must be the full `ssh-ed25519 AAAA... user@host` string including the key type prefix

### Agent container exits immediately

```bash
docker logs beszel-agent
```

Most common cause: malformed `BESZEL_KEY`. Ensure the full key string was copied without line breaks.

### Metrics show container stats but not host stats

The agent must run with `network_mode: host`. If it was accidentally deployed on a bridge network, update the compose in Portainer to match Section 1 and redeploy.

### Monitoring additional disks

Uncomment and adjust the extra-filesystems mount in the agent's volume list:

```yaml
- /mnt/disk/.beszel:/extra-filesystems/sda1:ro
```

The label after the colon (`sda1`) is what appears in the Beszel UI. Add one line per partition.
